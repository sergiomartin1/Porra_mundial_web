# 🌐 Cómo hacer la Porra accesible en Internet

La porra está lista para usarse. Aquí tienes **3 opciones gratuitas** para que todos puedan acceder desde cualquier lugar:

---

## ✅ Opción 1: GitHub Pages (Recomendada - Gratis y Fácil)

### Pasos:
1. **Crea una cuenta en GitHub** (si no tienes): https://github.com
2. **Crea un repositorio nuevo**:
   - Nombre: `porra-mundial-2026`
   - Público
   - Sin README
3. **Sube el archivo**:
   - Click en "uploading an existing file"
   - Arrastra `porra_mundial.html`
   - Commit
4. **Activa GitHub Pages**:
   - Ve a Settings > Pages
   - Source: Deploy from a branch
   - Branch: main / root
   - Save
5. **¡Listo!** Tu web estará en: `https://TU-USUARIO.github.io/porra-mundial-2026/porra_mundial.html`

**Ventajas:**
- ✅ Gratis para siempre
- ✅ Actualización fácil (sube el archivo cuando lo modifiques)
- ✅ URL permanente
- ✅ Funciona perfecto con localStorage

---

## ✅ Opción 2: Netlify Drop (Más Rápido - Solo arrastrar)

### Pasos:
1. Ve a: https://app.netlify.com/drop
2. Arrastra el archivo `porra_mundial.html` a la página
3. **¡Listo!** Te da una URL tipo: `https://nombre-random.netlify.app`

**Ventajas:**
- ✅ Inmediato (30 segundos)
- ✅ No necesitas cuenta
- ✅ Gratis

**Limitación:**
- ⚠️ Para actualizar el sitio necesitas crear cuenta en Netlify

---

## ✅ Opción 3: Vercel (Alternativa similar a Netlify)

### Pasos:
1. Ve a: https://vercel.com
2. Crea cuenta (con GitHub, Google o email)
3. "Add New Project"
4. Sube el archivo `porra_mundial.html`
5. Deploy

**Ventajas:**
- ✅ Gratis
- ✅ Muy rápido
- ✅ Actualizaciones fáciles

## ⚙️ Cómo funciona el almacenamiento

La web usa **localStorage** del navegador:
- ✅ Cada navegador guarda sus propios datos
- ✅ Las apuestas de cada usuario se guardan localmente
- ✅ Para compartir datos entre dispositivos, necesitas logearte en cada uno
- ⚠️ Si limpias las cookies/datos del navegador, se pierden las apuestas

**Importante:** El ADMIN debe introducir los resultados desde UN SOLO dispositivo/navegador y luego los demás podrán verlos al acceder desde sus dispositivos.

---

## 🔧 Si quieres una solución más avanzada (con base de datos compartida)

Para que todos vean las mismas apuestas en tiempo real desde cualquier dispositivo, necesitarías:
- Backend con base de datos (Firebase, Supabase, etc.)
- Esto requiere programación adicional

**La solución actual funciona perfectamente si:**
- Cada usuario hace sus apuestas desde su propio dispositivo
- El ADMIN introduce los resultados desde un dispositivo
- Los usuarios refrescan la página para ver los resultados actualizados
